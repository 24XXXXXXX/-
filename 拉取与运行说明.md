# 社区运动场地管理系统 - 拉取与运行说明

## 一、为什么启动前后端后看不到视频？

**原因：** 项目中的视频文件体积较大（部分超过 100MB），受 GitHub 单文件大小限制，**未将视频上传到 Git 仓库**。因此从本仓库拉取代码后，`communitySport/src/main/resources/static/upload/` 下只会存在**空目录结构**（通过 `.gitkeep` 保留），**不会有视频或上传的图片文件**。

- 拉取代码后：页面中依赖这些视频/图片的功能会显示为空或占位，属于**正常现象**。
- 如需本地看到视频效果：可在对应目录（如 `upload/video/`）下自行放置视频文件，或使用系统上传功能重新上传。

## 二、目录结构说明

以下目录在仓库中**仅保留空目录 + `.gitkeep`**，实际媒体文件不随代码提交：

- `communitySport/src/main/resources/static/upload/`（上传根目录）
- `upload/photo/`（头像、轮播、教练、场地等图片）
- `upload/video/`（视频）
- `upload/temp/`（临时文件）

程序会正常使用这些路径；若目录不存在，可参考仓库内已有结构自行创建。

## 三、如何启动项目

1. **后端**：在 `communitySport` 目录下使用 Maven 启动 Spring Boot 应用。
2. **前端**：在 `webFront` 目录下执行 `npm install` 后执行 `npm run dev`。

详细步骤可参考各子项目内的说明或 README。
