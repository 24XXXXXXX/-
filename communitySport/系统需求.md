# 社区运动场地管理系统（communitySport）系统需求（最终验收口径）

> 说明：本文件用于最终验收与后续维护。
>
>- 原 `系统需求.txt` 的功能点全部纳入。
>- 涉及 **支付** 与 **物流** 的部分，按原 `项目改造.md` 的口径固化为最终需求（钱包支付 + 简化物流）。
>- 文中标注为“增强需求”的能力：用于区分优先级，但**仍属于全量验收范围**（是否实现以《开发完成度.txt》为准）。

## 1. 角色与范围

系统包含 4 类角色：

- 用户（USER）
- 体育教练（COACH）
- 员工（STAFF）
- 管理员（ADMIN）

## 2. 通用技术与安全要求

- 后端基于 Spring Boot。
- 使用 Spring Security.
- 密码使用 BCrypt 加密存储（BCryptPasswordEncoder）。
- 认证方式：JWT（AccessToken + RefreshToken）。

## 3. 支付与资金（最终口径）

### 3.1 钱包支付（替代第三方支付）

系统内所有“需要在线支付”的业务（场地预约、课程预约、器材订单、视频购买等）统一使用钱包扣款：

- 当余额不足时应拒绝支付/下单。
- 成功扣款应写入资金流水（wallet_transaction）。

### 3.2 充值申请（管理员审核发放）

- 用户/教练/员工可提交充值申请.
- 管理员审核通过后发放到钱包.
- 申请金额范围：1-9999。

### 3.3 签到奖励

- 每日签到奖励：100.
- 连续签到 7 天：额外奖励 300（第 7 天发放）。

## 4. 物流（最终口径）

器材订单物流不接第三方接口，采用“简化物流状态”满足业务展示：

- 订单可维护发货信息（如快递公司/单号等可选字段）。
- 状态流转至少包含：已创建/已支付/已发货/已收货/已取消（及可选退款）。
- 用户可确认收货.

## 5. 用户端需求

### 5.1 用户注册登录

- 支持用户注册：填写用户名、密码等基础信息，校验通过后创建账号.
- 支持账号密码登录.
- 支持退出登录.
- 支持刷新 token.

增强需求：

- 密码找回（手机验证码或邮箱验证重置）。

### 5.2 系统首页推荐

首页需支持展示与跳转：

- 热门运动场地
- 优质教练课程
- 限时器材优惠
- 社区运动活动公告

增强需求：

- 基于用户历史预约/购买记录的个性化推荐。

### 5.3 运动场地预约

用户可：

- 查看社区内所有运动场地的详细信息：位置、类型、规格、开放时间、收费标准.
- 查看剩余可预约时段.
- 选择场地及时段提交预约并完成钱包支付.
- 生成预约凭证（核销码/凭证号等）。
- 在预约生效前取消预约并按规则退款.

筛选要求：

- 支持按场地类型筛选.
- 增强需求：按使用时段、价格等组合筛选.

### 5.4 体育器材购买

用户可：

- 浏览器材列表（名称、规格、用途、价格、库存数量）。
- 查看器材详情及用户评价.
- 加入购物车并批量购买.
- 选择或填写收货地址后完成钱包支付.
- 在订单中查看物流状态.
- 收货后可评价.

备注（地址口径）：

- 系统提供收货地址管理；下单时可以“选择地址”或“填写地址文本”作为最终收货信息.

### 5.5 体育教练课程预约

用户可：

- 查询教练资质信息、擅长领域、课程安排及用户评价.
- 查看课程详情：时长、内容、价格、上课场地、招生人数.
- 选择课程并预约，钱包支付后生成课程凭证.
- 查看课程预约状态.
- 课程结束后对教练及课程进行评分评论.

增强需求：

- 上课提醒（消息推送/站内信等）。

### 5.6 用户投诉

用户可：

- 提交投诉申请并选择类型：场地问题、器材质量、课程服务、其他.
- 填写投诉内容并上传凭证（图片/文字）。
- 在个人中心查看处理进度与反馈结果.
- 支持对处理结果进行二次反馈（追加沟通）。

### 5.7 个人中心

用户可：

- 修改个人基础信息：昵称、头像、联系方式、密码.
- 管理收货地址：添加、编辑、删除、设置默认.
- 查看个人预约记录、订单记录、投诉记录、消费明细.
- 管理个人收藏：收藏场地、教练、课程（以及可扩展到器材/视频）。

## 6. 体育教练端需求

### 6.1 注册与资质认证

- 普通用户可提交教练认证申请：姓名、联系方式、专业领域、资质证明材料.
- 管理员审核通过后开通服务权限（赋予 COACH 权限）。

### 6.2 课程管理

教练可：

- 发布课程信息：名称、类型、时长、价格、上课场地、招生人数、课程大纲.
- 编辑课程详情.
- 调整上课时间与价格（通过排期/场次维护）。
- 下架过期或满员课程.

### 6.3 预约处理

教练可：

- 查看用户课程预约通知与详情.
- 接单或拒单（拒单需理由）。
- 课程结束后确认核销状态.

增强需求：

- 课程开始前向用户发送上课提醒.

### 6.4 教练个人中心

教练可：

- 更新个人资料及资质信息.
- 展示个人评分与评价.
- 查看预约记录、历史订单与收入明细.

增强需求：

- 提现申请.
- 教练消息中心（接收用户咨询、系统通知）。

### 6.5 互动反馈

增强需求：

- 回复用户对课程的咨询与反馈.

## 7. 员工端需求

员工可：

- 处理用户投诉/咨询请求并跟进.
- 核销：运动场地使用、课程上课核销，记录核销时间与结果.
- 定期检查场地状态并更新（正常、维护中、故障）。
- 上报场地维护/维修申请.
- 核对器材库存，上报短缺/损坏，协助入库登记.
- 查看负责区域内基础数据并生成日报表.

增强需求：

- 巡检维护上报的完整闭环（工单/审批/关联资源）。
- 日报统计/可视化.

## 8. 管理员端需求

管理员可：

- 用户管理：查询、创建、编辑、冻结账号；查看预约/消费/投诉历史；处理账号申诉.
- 教练管理：审核资质、查看课程记录与评分反馈；对违规教练处理；下架不合规课程.
- 员工管理：新增、编辑员工账号；分配管理区域与职责权限；查看员工工作数据.
- 资源管理：
  - 场地信息（类型、位置、开放时间、收费标准）维护.
  - 场地预约规则配置（单次预约最长时长、可预约提前天数等）。
  - 器材信息维护（录入、编辑、价格、库存）。
- 系统管理：发布公告、管理轮播图与推荐内容；配置基础参数（支付方式、退款规则、投诉时限）；备份系统数据.
- 数据监控：查看使用率、用户活跃度、交易总额、诉求处理率等核心指标并生成报表.

增强需求：

- 系统基础参数管理、数据监控报表的完整后台可视化.
